# 实验二：交通灯控制

本文件夹用于归档实验二的最终程序、应试精简版程序和烧录文件。

## 功能说明

- 实现十字路口交通灯状态切换。
- 正常流程：横向通行 10 秒 -> 黄灯 5 秒 -> 竖向通行 10 秒 -> 黄灯 5 秒，循环运行。
- 使用定时器 0 中断计时，每 10ms 中断一次，累计 100 次得到 1 秒。
- 数码管显示当前状态剩余秒数。
- 拨码开关可临时强制切换状态：
  - 开关 1：强制横向通行。
  - 开关 2：强制黄灯。
  - 开关 3：强制竖向通行。
- 开关拨上后暂停定时器，灯切到目标状态，倒计时数字保持。
- 开关拨下后恢复拨上前的灯状态，并从原数字继续倒计时。

## 接线说明

- `P3.0` 连接数码管显示模块 `DATA`。
- `P3.1` 连接数码管显示模块 `CLK`。
- `P1.2` 控制横向通行灯组：LED4、LED7、LED12、LED15。
- `P1.3` 控制黄灯灯组：LED5、LED8、LED11、LED14。
- `P1.4` 控制竖向通行灯组：LED6、LED9、LED10、LED13。
- `P1.5`、`P1.6`、`P1.7` 分别对应拨码开关 1、2、3。

## Debug 说明

在 Keil Debug 中打开 `Peripherals -> I/O-Ports -> Port 1`。
P1 口窗口中，打勾表示高电平 1，取消勾表示低电平 0。
本实验拨码开关为低电平有效，所以取消 `P1.5`、`P1.6`、`P1.7` 的勾即可模拟开关拨上。

建议把断点设置在主循环的开关读取代码处：

```c
key = readKey();
```

或精简版中的：

```c
keyProcess();
```

程序运行到断点后，再修改 P1 口对应位，继续单步即可看到进入对应功能代码。
